# Переходы состояний персонажа

Сервер теперь использует явные правила переходов между состояниями персонажа.
Проверка выполняется в доменном слое перед запуском движения и перед явной
сменой состояния.

## Поддерживаемые состояния

- `standing` - персонаж стоит;
- `walking` - персонаж идет;
- `running` - персонаж бежит;
- `sitting` - персонаж сидит.

## Допустимые переходы

- из `standing` можно перейти в `standing`, `walking`, `running`, `sitting`;
- из `walking` можно перейти в `standing`, `walking`, `running`;
- из `running` можно перейти в `standing`, `walking`, `running`;
- из `sitting` можно перейти только в `standing`.

## Правила движения

- движение в состоянии `walking` выполняется со скоростью `50%` от базовой;
- движение в состоянии `running` выполняется со скоростью `100%` от базовой;
- попытка начать движение из `sitting` игнорируется;
- завершение движения всегда переводит персонажа в `standing`.

## Блокировка смены состояния

- после перехода `standing -> sitting` включается блокировка смены состояния на
  время из настройки `STATE_CHANGE_COOLDOWN_SECONDS`;
- после перехода `sitting -> standing` включается такая же блокировка на `5`
  игровых секунд;
- пока блокировка активна, сервер игнорирует любые команды, которые требуют
  смены текущего состояния игрока;
- блокировка затрагивает и команды движения, потому что запуск движения меняет
  состояние на `walking` или `running`.

## Поведение при недопустимом переходе

Если клиент отправляет команду, которая нарушает матрицу переходов, сервер не
меняет состояние игрока и не публикует событие.
