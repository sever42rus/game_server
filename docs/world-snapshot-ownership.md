# Ответственность за снимок мира

`WorldState` хранит доменное состояние мира и не формирует данные клиентского
протокола. Это сохраняет доменный слой независимым от текущего состава
набора пакетов начальной синхронизации клиента.

Снимок мира собирает `GameService`. Сейчас в `WorldSnapshotDTO` передаются
игроки и выбранная цель каждого игрока через поле `selected_target`. Поле
присутствует всегда: если цель не установлена, значение равно `null`. На уровне
инфраструктуры этот DTO преобразуется в поток пакетов `OWN_PLAYER`,
`PLAYER_SPAWN` и `SNAPSHOT_END`. Новые группы сущностей, например NPC, боты или
объекты мира, должны добавляться в DTO на уровне приложения.

Выбранная цель хранится в домене как расширяемая ссылка с типом цели. На данный
момент поддерживается тип `player`; клиент может выбрать игрока через команду
`select_target` и сбросить цель через `clear_target`.

Успешный старт движения игрока также сбрасывает его выбранную цель. Сброс
публикуется исходящим пакетом `PLAYER_TARGET_CHANGED` без выбранной цели, после
чего последующие начальные пакеты `PLAYER_SPAWN` также содержат `null` у этого
игрока на уровне DTO.

Такой подход оставляет `WorldState` контейнером доменных сущностей, а решение о
том, какие сущности и в каком виде получает клиент, остается в сценарии
подключения игрока.
