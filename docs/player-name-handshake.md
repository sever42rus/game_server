# Обязательное имя перед входом в мир

Сервер больше не добавляет клиента в игровой мир сразу после открытия TCP-соединения.
Сначала клиент обязан отправить пакет `SET_NAME` с именем игрока.

## Новый порядок входа

1. Клиент открывает TCP-соединение.
2. Сервер создает транспортную сессию и ожидает первый пакет `SET_NAME`.
3. Клиент отправляет имя игрока в UTF-8.
4. Сервер проверяет имя и только после этого добавляет игрока в `WorldState`.
5. Новый клиент получает `OWN_PLAYER`, набор `PLAYER_SPAWN` и `SNAPSHOT_END`.
6. Остальные клиенты получают `PLAYER_SPAWNED` уже с именем нового игрока.

## Ограничения имени

- имя обязательно;
- пустое имя и имя из одних пробелов отклоняются;
- используется кодировка `UTF-8`;
- максимальная длина имени — `24` байта после кодирования в `UTF-8`.

## Передача имени клиентам

Имя игрока включается:

- в каждый пакет `PLAYER_SPAWN` во время начального снапшота;
- в пакет `PLAYER_SPAWNED` при появлении нового игрока после инициализации.

В пакеты `PLAYER_UPDATED` и `PLAYER_MOVE_STARTED` имя не добавляется, потому что
оно не меняется на каждом тике и не должно увеличивать постоянный сетевой трафик.
